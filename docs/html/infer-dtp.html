InferDPT & RANTEXT
— what governs the privacy–utility outcome

InferDPT performs privacy-preserving inference with a closed-box remote model: the client perturbs its prompt under token-level local differential privacy (RANTEXT), sends the perturbed prompt to the remote model, and reconstructs an aligned output with a local model. This page measures the scheme through three design levers, the privacy budget ε, the embedding map φ, and the local extraction model X, with attack-independent leakage probes (mutual information, plausible deniability, PII survival) paired against reconstruction attacks (embedding inversion, masked-LM inference). The result: on a modern sentence-embedding space the perturbation mechanism reduces to near-uniform substitution at every ε, because the embedding map, not ε, sets the operating point.

§01

Introduction

the problem, the threat model, the headline number

RANTEXT replaces every in-vocabulary token of a document with a semantically near token sampled under ε-local differential privacy, so the perturbed prompt sent to a remote model reveals little about the original. Whether this preserves utility depends on the geometry of the embedding map φ that defines "near". We find that on a modern sentence-embedding space the mechanism collapses: at ε=1 the per-token mutual information between a raw token and its replacement is 0.0 bits, and at every ε the replacement cosine sits at the random-pair floor (0.61), i.e. the replacement is statistically a random token.

Threat model. The remote model is honest-but-curious. The adversary knows the mechanism, the vocabulary V, the embedding map φ, and the budget ε, and attempts to recover the raw tokens from the perturbed text it receives. The local extraction model is trusted and never leaves the client.

Three levers. The privacy–utility outcome is set by (1) the budget ε, which is intended to trade privacy for utility but is largely inert here, (2) the embedding map φ, which sets whether "near" is meaningful and is the binding constraint, and (3) the local extraction model X, which recovers utility downstream and does not affect privacy. The headline finding is that φ dominates: substituting an isotropic LLM input-embedding matrix for the sentence-embedding map removes the anisotropy but lowers perturbation faithfulness further (replacement cosine 0.23 against 0.79), because its distances are even more concentrated.

Preliminaries

S_w / N_wPlausible-deniability statistics (Feyisetan et al.): S_w = P[M(w)=w] (self-substitution, lower is more private); N_w = number of distinct outputs over repeated runs (higher is more private).

replacement similarity (retention)Mean cosine between an original token and its sampled replacement (0.61 = random baseline, 1.0 = unchanged). Equal to the random baseline means no semantic faithfulness.
anisotropyMean cosine of random token pairs. 0 = isotropic; large = all vectors share a common direction, which compresses the distance range.
PII semantic leakage / reconstruction recallPresidio detects PII spans in Doc; the probe scores each entity's maximum cosine to the target text. On Doc_p (lower is more private); on the final output (higher is more faithful).
embedding inversionAttack: for each perturbed token return the top-K nearest vocabulary tokens in φ; a hit means the raw token is recovered. recovery@K.

§03

Method

InferDPT pipeline, RANTEXT mechanism, and the measurement loop

InferDPT (three modules). (1) The perturbation module tokenises Doc, and for each in-vocabulary token samples a replacement under ε-LDP, producing Doc_p; out-of-vocabulary tokens are discarded. (2) The remote model returns Gen_p = generate(instruction ∥ Doc_p). (3) The extraction module, a local trusted model X, is given the raw Doc as prefix and Gen_p as material, and writes a continuation aligned to Doc. Privacy holds because the remote model only ever sees Doc_p; utility is recovered because X re-grounds on the true Doc.

RANTEXT (per token). Embed the token, eb_t = φ(x). Draw a Laplace noise vector and set the radius to its norm. The candidate set C_r is every vocabulary token whose embedding lies within that radius of eb_t. Score each candidate u = 1 − d/radius and sample one with probability proportional to exp(ε/2 · u). The radius is random per token (the "random adjacency list"), and ε controls both the radius scale and the sampling sharpness.

Measurement loop. For each lever setting we (1) run the full pipeline on a development corpus, (2) compute attack-independent probes on the perturbation and on the outputs, (3) run reconstruction attacks on Doc_p, and (4) read the privacy and utility together. Implementation: src/inferdpt/{rantext,probes,attacks,diagnostics,eval}.py; remote and extraction calls through one OpenAI-compatible client; φ is a cached vocabulary-embedding matrix. The corpus is small (5 documents) and single-seed, so quantitative claims below are preliminary.

§05

Results

by design lever: ε, embedding map φ, extraction model X

§5.1 — Lever 1: privacy budget ε

Embedding map: pythia-410m input-embedding matrix (isotropic, anisotropy 0.011), 12k cl100k tokens, cased CNN/Daily Mail (N=60). The setting that controls how aggressively a token is replaced is the noise radius, set by the calibration factor Z(ε). The reference RANTEXT constants for Z are a curve-fit to one token ("happy") in OpenAI's ada-002 space (paper Appendix B), and they mis-size the radius on any other embedding — applied to this map they saturate the candidate set and ε goes inert. We instead refit Z per embedding to a fixed candidate-set target |C_r|/|V| ≈ 1% (the paper's own calibration method, not a per-model fudge of a secondary quantity), so the operating point is held constant and ε is isolated as the only moving knob. Grid ε ∈ {2, 6, 10, 14}.

Toggle the embedding map above the chart to switch to ada-002 (the paper's anisotropic φ) at the same calibrated |C_r|: the same ε-monotone trends hold, but its anisotropy depresses both perturbation-side coherence and the inversion attack (lower inv@10), the coherence↔privacy trade documented in §5.2.

Conclusion. With the radius calibrated to the embedding (not the ada-002 constants), ε is a genuine, monotone lever. Leakage rises steeply with ε — overlap 0.22 → 0.78, self-substitution S_w 0.20 → 0.66, PII-leak recall 0.13 → 0.57, token leakage 4.6 → 5.2 bits, embedding inversion@10 0.67 → 0.99 across ε 2 to 14 — while the candidate set is held at ~1% throughout, so this is ε acting, not a radius artifact. Perturbation-side fidelity rises in step (replacement cosine 0.51 → 0.97, Gen_p coherence 0.54 → 0.74, u_p_control 0.55 → 0.76): higher ε keeps more of the document's meaning in Doc_p. The final extracted output, by contrast, stays faithful at every ε (reranker utility and PII reconstruction recall ~1.0, SimCSE utility 0.90 → 0.93), because the trusted extraction model re-grounds on the true Doc. So ε trades privacy for perturbation-side fidelity along a clean Pareto curve; because the final output is ε-robust, the practical choice is to run ε as low as the downstream task tolerates. This corrects the earlier reading that ε is inert — that was an artifact of the ada-002 noise constants applied to a different embedding, not a property of the mechanism.

§5.2 — Lever 2: embedding map φ (anisotropy & the noise radius)

φ enters RANTEXT only through Euclidean distances, so its one structural property is the distance scale. Anisotropy — a large shared direction in every token vector — compresses that scale into a thin shell, with two consequences. First, the noise radius (the Laplace-norm threshold that defines the candidate set C_r, set by Z(ε)) must be matched to the shell. The paper's fixed Z is a curve-fit to one token in ada-002 (§5.1), so on any other φ it mis-sizes: too large and C_r becomes a big slice of V, the sampling goes near-uniform, and Doc_p is word-salad. Second, even at a matched radius the compressed shell blurs the candidate ranking, which trades Doc_p coherence against the inversion attack.

1 · The noise radius is the control. The mechanism's real knob is the size of the noise radius — equivalently the candidate-set fraction |C_r|/V it admits. As the radius shrinks (pythia-410m, the prompt below, ε=6):

Conclusion. φ is not an end-to-end utility lever (final utility is flat, extraction-dominated). Its one established effect is (a) a calibration constraint — the noise radius must be refit per φ or the mechanism degenerates to salad. The apparent (b) coherence↔leakage trade across ada-002 vs pythia-410m turns out to be a confound, not anisotropy: the controlled test — the same qwen matrix raw (aniso 0.61) vs mean-centered (aniso 0.00), identical vocabulary and matched realized |C_r|, varying only anisotropy — does not reproduce it. At matched |C_r| (ε=2/6/10) centering leaves coherence and utility unchanged and, if anything, lowers inv@10 (−0.14 to −0.24) and PII-leak — the opposite sign from the ada-vs-pythia comparison. So the ada-vs-pythia trade was driven by vocabulary coverage (ada drops names → less PII), effective dimension, and embedding family — not by anisotropy. Anisotropy matters only through (a): it sets the distance scale the radius must be matched to. (N=10, single seed — directional; firm with N≥30 + seeds + a dose-response.)

§5.3 — Lever 3: extraction model X (why the headline utility is misleading)

X is the trusted local model, and it is handed the true Doc as prefix — only Gen_p comes from the perturbed prompt. So X re-grounds on the real document and reconstructs its content no matter how badly ε or φ damaged Doc_p. That makes end-to-end utility a poor gauge of the perturbation: it stays ~0.9 even when Doc_p is word-salad, because most of it is X working from the clean prefix — not privacy-preserving remote inference. The extraction model therefore disguises the two upstream levers and creates a false sense that the private channel is doing the work.

Conclusion. X is a utility-only lever that masks the perturbation: the headline utility mostly measures the trusted local model re-reading the true Doc, barely moves with ε or φ, and overstates what the perturbation + remote channel actually contribute (+0.05–0.11). The honest privacy↔utility question lives at Doc_p — coherence and leakage, upstream of X — not in the end-to-end number.

§06

Findings

claims · grounds · scope (each judged by an external cross-model gate)

1 · The published mechanism does not reproduce its own Table II; Z(ε) is a utility knob, not a privacy quantity (Supported). Run on the paper's own ada-002 embeddings with the published Z(ε), the realized candidate set is 13–39% of the vocabulary and Doc_p is word-salad at every ε up to 14. Theorem 2's proof (Appendix C) is the plain exponential-mechanism argument — Δu=1, 0<u≤1 ⇒ ratio ≤ eᵋ — in which Z, the radius, and the noise never appear; Z is a scipy.curve_fit to the single token "happy" in ada-002 (Appendix B). Grounds: §5.1, rantext.md. Scope: assumes our reimplementation faithfully matches the public repo.

2 · After per-φ radius calibration, ε is a genuine monotone privacy↔coherence lever, but end-to-end utility is ε-flat (Supported). Holding realized |C_r|≈1%, leakage (overlap, S_w, inv@10, PII-leak, token-MI) and Doc_p coherence rise monotonically with ε, while final utility barely moves (0.90→0.93). Grounds: §5.1. Scope: the lever acts on the perturbation-side / privacy observables; the final number is not responsive.

3 · φ is not a utility lever, and anisotropy does not explain the φ tradeoff (Partial). Final utility is flat (~0.9) across every φ. The coherence↔leakage difference seen across embeddings (ada-002 vs pythia) is not reproduced by the controlled same-matrix raw-vs-centered test — coherence stays flat and the leakage signs reverse — so anisotropy is not sufficient to cause it. Attributing the cross-embedding gap specifically to vocabulary coverage / effective dimension / family is plausible but not yet proven. Grounds: §5.2. Scope: control is N=10, single seed, within-qwen (ε=14 |C_r| unmatched) — the least settled claim here.

4 · The extraction model disguises the perturbation (Supported). A local-only baseline — the trusted model given the true Doc with no remote output — already reaches ~0.83 of the ~0.90 end-to-end utility; the private remote channel adds only +0.05–0.11 (smaller at lower ε), and its raw output (u_p_control) is degraded (0.55–0.76). So the headline utility mostly measures the local model, not private inference. Not claimed: that InferDPT is worse than local-only — the marginal is positive on cos-to-Doc. Grounds: §5.3.

5 · What is protectable is set by the vocabulary, not ε (Supported). With cl100k BPE + cased text a common name is a single token; whether it is dropped (privacy-by-removal) or substituted is decided by vocabulary coverage (ada's 10129 drops names, our 12000 substitutes), independent of ε. Grounds: §5.2.

§07

Analysis

why each finding holds — the mechanism behind the numbers

Why the radius must be recalibrated, and why ada-002 salads. φ enters RANTEXT only through Euclidean distances, and the noise radius = ‖Laplace(Δφ/Z)‖. Z(ε) is fitted to ada-002's "happy" distance scale and saturates for ε≥2, so on any embedding whose distance shell differs the radius over- or under-shoots. On ada-002 itself the radius (~0.6) exceeds the anisotropic inter-token distance (nearest-neighbour ~0.5), so C_r swallows a large slice of V and the score u = 1 − d/radius flattens → near-uniform sampling → salad. Re-fitting Z per φ to a |C_r| target restores a graded candidate set.

Why ε works once |C_r| is pinned. With the radius calibrated the candidate set is fixed, so ε drives only the exponential sharpness exp(ε/2·u): higher ε concentrates probability on the nearest tokens → smaller change → higher coherence and higher leakage. That is why the tradeoff becomes a clean monotone curve only after calibration — ε is a sharpness dial at a fixed operating point, not a radius dial.

Why anisotropy is not the lever. Anisotropy's only role is to set the distance scale — which the radius calibration already absorbs. With |C_r| matched, toggling anisotropy (mean-centering the same matrix) leaves coherence and utility unchanged and slightly lowers leakage; the cross-embedding differences (ada vs pythia) therefore trace to vocabulary coverage and dimensionality, not the shared-direction geometry. inv@10 is consequently not a clean cross-φ privacy metric.

Why extraction dominates, and what MI bounds. X is handed the true Doc, so the final output is mostly the local model re-grounding on the real prefix; the private channel contributes a shrinking increment. On privacy, the per-token channel p(y|x) is analytic, so I(token;replacement) upper-bounds per-token recovery by Fano — but RANTEXT is memoryless while the source is correlated (I(x_i;Y_{1..n}) ≥ I(x_i;Y_i)), so a context-aware LM adversary can exceed it. The measured inv@10 / masked-LM attacks are weak lower bounds, not the worst case. See docs/research/mi-probes.md.

§08

Discussion

what the scheme buys, design implications, limitations, next

What the scheme actually buys. At a fair operating point the privacy↔coherence tradeoff is tunable by ε at the Doc_p level, but the end-to-end utility is dominated by the trusted local extraction model, so the headline utility overstates the value of the private remote channel (a +0.05–0.11 increment over local-only). The honest privacy↔utility question therefore lives at Doc_p — coherence and leakage — not in the final number.

Design implications. (a) Z must be calibrated per embedding; the paper's fixed constants do not transfer and silently degenerate. (b) φ choice is second-order — pick one whose distance scale is calibratable; anisotropy/isotropy is not a privacy dial. (c) Always report a local-only baseline when claiming utility, or you are measuring X, not the mechanism. (d) What is protectable is governed by the vocabulary and OOV handling, not by ε — RANTEXT has no sensitivity detector, so in-vocabulary sensitive content leaks at any utility-preserving ε.

Limitations. Single seed throughout; N=60 for the main sweeps, N=10 for the anisotropy control (within-qwen, ε=14 |C_r| unmatched). MAUVE is invalid at this corpus size (k-means needs ≥234 points) and is excluded. Utility is SimCSE cos(Doc, output), a proxy — a task-level metric could expose effects (e.g. context contamination) that cosine misses. inv@10 assumes the adversary knows φ and is weak relative to a context-aware LM reconstruction. The non-reproducibility finding assumes our reimplementation matches the public repo.

Next. N≥30 with multiple seeds and CIs for the anisotropy control, plus a dose-response (subtract α·mean for several α); a context-aware LM reconstruction attack (the real worst case) and a task-level utility metric, both reported against the local-only baseline.
